## 7-1 좋은 지시 = 좋은 분석

### 도입
- AI에게 데이터 분석을 시키면 결과는 나오지만, **"좋은 지시"를 해야 "좋은 결과"**가 나옴. 같은 AI라도 **지시문(프롬프트)**에 따라 결과 품질이 달라짐.

### 나쁜 지시 vs 좋은 지시
- **Before**: "이 데이터 분석해줘" → 방향 없는 기본 통계만 나열.
- **After**: "고등학교 1학년 200명 설문에서 **수면시간과 성적만족도의 관계**를 분석해줘. 산점도 그리고, 상관계수 계산하고, 3줄 요약해줘. **상관관계를 인과관계로 해석하지 말아줘.**" → 목표에 맞는 분석·시각화·해석.

### 좋은 지시문의 5가지 요소
| 요소 | 설명 | 예시 |
|------|------|------|
| **배경/맥락** | 분석의 목적과 상황 | "고등학교 1학년 200명 설문 데이터를 분석하려 합니다" |
| **데이터 설명** | 데이터 구조와 변수 | "변수: 학년, 성별, 수면시간, 운동일수, 스마트폰사용, 성적만족도 등 9개" |
| **분석 과제** | 구체적으로 무엇을 하라는지 | "수면시간과 성적만족도의 관계를 분석해줘" |
| **결과 형식** | 결과를 어떤 형태로 원하는지 | "산점도 차트 + 상관계수 + 해석 3줄 요약" |
| **조건/제약** | 주의사항·제한조건 | "상관관계를 인과관계로 해석하지 말고, 제3변수 가능성도 언급해줘" |

- 5요소가 모두 포함되면 AI가 **정확한 분석·적절한 시각화·올바른 해석**을 할 확률이 높아짐. **조건/제약**을 명시하면 인과관계 오해 등 흔한 실수를 예방할 수 있음.

### 나쁜 지시문 예와 개선
- "데이터 분석해줘" → 데이터·분석 목표·결과 형식 모두 없음 → 구체적으로 데이터·목표·형식 명시.
- "뭔가 재미있는 거 찾아줘" → 주관적·모호 → (1) 월별 매출 추세, (2) 상위 5개 품목, (3) 요일별 패턴 등 **구체적 분석 목표** 제시.
- "차트 만들어줘" → 데이터·차트 종류·형식 없음 → "2024년 월별 기온으로 꺾은선 그래프, x축 월, y축 평균기온(°C), 제목 …" 등 명시.

## 7-2 소울 문서 작성하기

### 소울 문서(Soul Document)란?
- AI에게 **분석 프로젝트의 목적, 데이터, 방법, 원칙, 결과 형식**을 체계적으로 전달하는 문서. **일회성 질문**이 아니라 **프로젝트 단위**의 깊은 분석에 특히 효과적.

### 소울 문서의 5개 섹션
| 섹션 | 내용 | 예시(플레이스홀더) |
|------|------|---------------------|
| **역할 정의** | AI가 맡을 전문가 역할 | "너는 데이터 분석 전문가이다. 고등학생이 이해할 수 있는 수준으로 설명해야 한다." |
| **배경 정보** | 프로젝트 맥락·데이터 설명 | "200명 고등학생 설문(학년, 성별, 수면시간, 운동일수, 스마트폰, 성적만족도, 스트레스, 아침식사, 이동시간)을 분석한다." |
| **분석 과제** | 단계별 수행 작업 | "1단계: 데이터 개요 / 2단계: 주요 변수 간 상관 / 3단계: 그룹 비교(성별·학년) / 4단계: 핵심 발견 3가지" |
| **분석 원칙** | 반드시 지켜야 할 규칙 | "상관을 인과로 해석하지 않는다. 표본 크기·한계를 명시. 이상치 제거 시 근거를 밝힌다." |
| **결과물 형식** | 최종 산출물 구조 | "요약(3줄 이내), 시각화(산점도·막대그래프 각 1개), 상세 분석, 한계점·후속 연구 제안" |

- 한 번 만들면 **반복 사용** 가능. 같은 데이터로 여러 번 분석할 때 소울 문서를 조금씩 수정하며 대화하면 일관된 분석이 가능.

## 7-3 데이터 윤리

### 도입
- "이 데이터를 이렇게 써도 되는가?"를 묻는 것이 데이터 리터러시의 핵심 축. **데이터로 할 수 있는 것**과 **해도 되는 것**은 다름. 기술적으로 가능하다고 윤리적으로 올바른 것은 아님. **"누가 영향을 받는가?"**를 먼저 생각.

### 데이터 윤리의 5대 원칙
| 원칙 | 설명 | 예시 |
|------|------|------|
| **개인정보 보호** | 수집 전 동의, 꼭 필요한 정보만 수집 | 설문 시 개인 식별 불가능하게 익명화 |
| **공정성** | 특정 집단에 불리하지 않도록 편향 확인·제거 | 인종·성별·나이에 따른 차별 없는 분석 |
| **투명성** | 분석 방법·한계를 솔직하게 공개 | "이 결과는 200명 표본이므로 일반화에 주의" |
| **정직한 보고** | 원하는 결론에 맞게 데이터를 왜곡하지 않음 | 불리한 결과도 숨기지 않고 보고 |
| **사회적 책임** | 분석 결과가 사회에 미칠 영향을 고려 | 차별을 강화하는 활용 방지 |

### 실제 사례로 보는 데이터 윤리
- **Amazon 채용 AI (2018)**: 과거 데이터 학습 결과 여성을 체계적으로 불리하게 평가 → 폐기. → **공정성**.
- **Cambridge Analytica (2018)**: Facebook 사용자 8,700만 명 데이터가 동의 없이 정치 캠페인에 활용. → **개인정보 보호**.
- **COMPAS 재범 예측 (2016)**: 재범 예측 AI가 흑인에게 불공정하게 높은 재범 위험 부여. → **공정성, 투명성**.
- **코로나 접촉추적 앱 (2020)**: 감염병 예방 vs 프라이버시 침해 논란. → **개인정보 보호, 사회적 책임**.

### 시나리오 예: 학생 성적 예측 AI
- 이해관계자: 학생, 교사, 학부모, 학교.
- 이슈: 개인정보(동의 없이 수집 가능), 편향(저소득·다문화 학생이 불공정하게 "위험군" 분류), 자기실현적 예언(예측 라벨이 기대를 낮춰 실제 성적 하락), 투명성(AI 설명 불가 시 신뢰 어려움).

### 데이터 윤리에는 정답이 없는 경우가 많음
- "효율성 vs 프라이버시", "안전 vs 자유"처럼 **가치가 충돌**하는 상황에서 **균형점을 찾는 능력**이 중요. 핵심은 **"영향받는 사람의 입장"**에서 생각하는 것.

## 7-4 데이터로 소통하기

### 도입
- 훌륭한 분석도 **제대로 전달하지 못하면** 의미가 없음. 데이터를 **"이야기"로 만들어 전달하는 능력**이 데이터 리터러시의 마지막 퍼즐.

### 데이터 스토리텔링 5단계
1. **핵심 메시지 정하기** — 한 문장으로 요약할 수 있는 핵심 발견.
2. **청중 파악하기** — 누구에게 전달하는가? (교사/학생/학부모 → 전문 용어 수준 결정)
3. **근거 차트 선택** — 메시지를 가장 잘 보여주는 시각화 (추세→꺾은선, 비교→막대, 관계→산점도).
4. **내러티브 구성** — 문제 → 발견 → 의미 → 제안 ("왜 궁금했나 → 무엇을 발견했나 → 의미 → 다음 행동").
5. **한계 명시** — 분석의 제한점을 솔직하게 밝히기 ("200명 표본이라 일반화에 주의", "상관이지 인과가 아님").

### So What?
- **"상관계수가 0.35이다"** → **"그래서 뭐?"** → **"수면 시간이 긴 학생이 성적 만족도가 높은 경향이 있다"**. 숫자를 넘어 **의미와 행동**으로 연결하는 것이 스토리텔링의 핵심.

### 나쁜 보고서 vs 좋은 보고서 (예: 수면과 성적)
- **나쁜 예**: 제목 "수면과 성적", 수치만 나열(r=0.35, p<0.01, 평균, 표준편차, 회귀식…) — 해석 없음, 청중 불분명, "그래서 뭐?"에 답 없음.
- **좋은 예**: 제목 "잠을 더 자는 학생이 성적에 만족할까?", 200명 조사 결과 요약 + **핵심 발견**(7시간 이상 79% 만족 vs 6시간 미만 45%) + **주의**(인과 아님, 역인과 가능성) + **제안**(수면 환경 개선 후속 조사 권고).

### 보고서 구성 요소 (실습)
- 제목, 핵심 발견, 근거(데이터/수치), 의미/해석(인과 주의), 한계점, 제안/다음 행동.

## 챕터 7 종합: 이번에 배운 것

- **7-1**: 모호한 지시 vs 구체적 지시의 결과 차이, 좋은 지시문 5요소(배경·데이터·과제·형식·조건), 나쁜 지시문 개선·직접 작성 연습.
- **7-2**: 소울 문서의 개념·필요성, 5개 섹션(역할·배경·과제·원칙·형식), 실전 예시·직접 작성 연습.
- **7-3**: 데이터 윤리 5대 원칙(개인정보·공정성·투명성·정직한 보고·사회적 책임), 실제 사례, "할 수 있는 것 vs 해도 되는 것", 영향받는 입장에서 생각하기.
- **7-4**: 스토리텔링 5단계, "So What?"—숫자를 의미로 변환, 청중에 맞게 메시지 조절, 구조화된 데이터 보고서 작성 연습.

## 레슨 과제

1. AI에게 데이터 분석을 지시할 때, '나쁜 지시'와 '좋은 지시'의 차이를 한 가지씩 들어보세요. 좋은 지시에는 어떤 요소가 들어가면 좋을까요?

2. 소울 문서의 다섯 가지 섹션(역할 정의, 배경 정보, 분석 과제, 분석 원칙, 결과물 형식) 중에서, '분석 원칙'에 "상관관계를 인과관계로 해석하지 않는다"를 넣는 이유를 적어보세요.

3. 데이터 윤리의 다섯 가지 원칙(개인정보 보호, 공정성, 투명성, 정직한 보고, 사회적 책임) 중 하나를 골라, 그 원칙을 지키지 않았을 때 생길 수 있는 문제를 예를 들어 설명해 보세요.

4. 데이터 스토리텔링에서 "So What?"에 답한다는 것은 무엇을 의미하나요? 숫자만 나열하는 보고와 비교해서 설명해 보세요.

5. "데이터로 할 수 있는 것"과 "해도 되는 것"이 다를 수 있다는 말의 의미를, 개인정보나 편향을 예로 들어 한 문장으로 적어보세요.

6. 'AI 활용과 데이터 윤리' 단원을 배운 뒤, AI로 분석을 요청할 때 꼭 넣고 싶은 '조건'이나 '원칙' 한 가지를 정해 보세요.
