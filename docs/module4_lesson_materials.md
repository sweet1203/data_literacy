# 모듈 4 수업자료: 데이터 탐색하기

실제 데이터를 직접 다루며 패턴과 이상치를 발견하는 모듈입니다.

| 레슨 | 제목 | 소요 시간 | 핵심 |
|------|------|----------|------|
| 4-1 | 데이터 읽기 기초 | 20분 | 행·열·셀, 정렬·필터, 테이블 탐색 |
| 4-2 | 기초 통계 이해하기 | 25분 | 평균·중앙값·표준편차 |

## 4-1 데이터 읽기 기초

### 도입
- 데이터 리터러시의 첫 실전 능력은 **데이터 테이블을 읽는 것**. 대부분의 데이터는 **행(row)**과 **열(column)**로 된 표 형태.

### 데이터 테이블 구조
- **행(row)**: 가로줄. 하나의 관측 대상(예: 학생 1명)에 대한 모든 정보.
- **열(column)**: 세로줄. 하나의 특성(예: 수면 시간)에 대한 모든 값.
- **셀(cell)**: 행과 열이 만나는 칸. 특정 대상의 특정 값.

### 사용 데이터: 200명 학교 설문
- 열 예: id, grade(학년), gender(성별), smartphone_hours, sleep_hours, fatigue(피로도), satisfaction(만족도), exercise_days, study_hours 등.
- 처음 볼 때는 숫자보다 **열 이름(변수)**를 먼저 확인 — "어떤 정보가 담겼는가?" 파악이 첫 단계.

### 정렬(Sort)
- **오름차순**: 작은 값 → 큰 값 (예: 수면 시간 4.5 → 5.0 → 6.2 …)
- **내림차순**: 큰 값 → 작은 값
- 정렬만으로도 "스마트폰 가장 많이/적게 쓰는 학생", "만족도 최고/최저 학생의 특징", "이상치 유무" 등을 파악 가능.

### 필터
- 특정 조건에 맞는 행만 보기 (예: 학년=1, 만족도 ≥ 7). 원하는 정보를 빠르게 찾는 데 사용.

## 4-2 기초 통계 이해하기

### 기초 통계량
- **대표값**: 데이터의 중심 (평균, 중앙값)
- **산포도**: 데이터가 얼마나 흩어져 있는지 (표준편차, 범위)

### 평균(Mean)
- 모든 값을 더한 뒤 개수로 나눈 값. **이상치에 민감**.
- 예: 5명 월급 200, 250, 300, 280, **5,000**만원 → 평균 1,206만원 (대표성 낮음).

### 중앙값(Median)
- 크기순으로 나열했을 때 **가운데** 값. **이상치에 강건(robust)**.
- 홀수 개: 가운데 값 / 짝수 개: 가운데 두 값의 평균.
- 위 월급 예: 중앙값 280만원 → 평균보다 현실적인 대표값.

### 언제 평균? 언제 중앙값?
| 평균이 적합 | 중앙값이 적합 |
|-------------|---------------|
| 데이터가 대칭적으로 분포 | 데이터가 한쪽으로 치우침 |
| 극단값 없음 | 극단값(이상치) 있음 |
| 시험 점수, 키, 체중 등 | 소득, 집값, 재산 등 |
| "전체 합계"가 중요할 때 | "전형적인 값"이 중요할 때 |

- 소득처럼 일부가 매우 높은 분포에서는 **중앙값**이 더 현실적.

### 표준편차(Standard Deviation)
- 각 값이 **평균으로부터 얼마나 떨어져 있는지**를 나타내는 수치.
- **작으면**: 데이터가 평균 근처에 모여 있음 (일관적).
- **크면**: 평균에서 멀리 퍼져 있음 (변동 큼).
- 예: A반 표준편차 작음 → 대부분 비슷한 점수 / B반 표준편차 큼 → 점수 차이 매우 큼.

### 통계량 해석
- 통계량은 **맥락 속에서 해석**해야 의미가 생김. "평균 7시간"보다 "권장 수면(8~10시간)보다 부족하다"처럼 해석하는 것이 데이터 리터러시.

## 4-3 패턴 찾기

### 데이터에서 찾는 3가지 패턴
1. **추세(Trend)**: 시간·순서에 따른 변화 방향 (증가? 감소? 유지?)
2. **비교(Comparison)**: 그룹 간 차이 (남 vs 여, 1학년 vs 3학년)
3. **상관관계(Correlation)**: 두 변수가 함께 변하는 경향 (A↑ → B↑?)

### 패턴 1: 그룹 간 비교
- 학년별 스마트폰 사용 시간처럼 **막대 차트**로 그룹별 평균 비교. "학년이 올라가면 사용이 늘어난다"는 **상관**이지 **인과**가 아님 — 다른 요인(자율성, 교우 관계 등) 가능성 있음.

### 패턴 2: 분포 비교
- **히스토그램**으로 성별 만족도 분포 비교. **봉우리(peak) 위치**와 **퍼짐 정도**를 보면 그룹 차이를 더 잘 이해할 수 있음.

### 패턴 3: 상관관계
- **양의 상관**: X↑ → Y↑ (예: 키와 체중)
- **음의 상관**: X↑ → Y↓ (예: 스마트폰과 수면)
- **상관 없음**: X와 Y 사이에 패턴 없음 (예: 신발 사이즈와 시험 점수)
- **산점도**가 상관관계를 보여주는 기본 도구. **"상관관계는 인과관계가 아니다"** — 아이스크림 판매와 익사 사고는 여름(제3변수)이 공통 원인.

### 패턴 4: 복합 비교
- 학년별 스마트폰·공부·운동 시간을 함께 비교하는 등 여러 변수를 동시에 보면 더 깊은 통찰 가능.

## 4-4 이상치 탐정

### 이상치(Outlier)란?
- 다른 데이터와 비교했을 때 **비정상적으로 크거나 작은 값**. **오류**일 수도 있고 **진짜 특이 사례**일 수도 있음. **발견 → 원인 파악 → 적절히 처리**가 중요.

### IQR 방법 (이상치 판별)
1. 데이터를 **크기순으로 정렬**
2. **Q1(제1사분위수)**, **Q3(제3사분위수)** 구하기
3. **IQR = Q3 - Q1** (가운데 50% 범위)
4. **하한 = Q1 - 1.5×IQR**, **상한 = Q3 + 1.5×IQR**
5. **경계 밖의 값 = 이상치**

- 1.5배는 존 튜키가 제안한 기준. 3배를 쓰면 극단 이상치만, 1배를 쓰면 더 많은 값이 이상치로 잡힘.

### 이상치가 미치는 영향
| 영역 | 영향 |
|------|------|
| **평균** | 이상치가 평균을 크게 왜곡 |
| **상관계수** | 이상치 하나가 상관을 만들거나 없앨 수 있음 |
| **표준편차** | 표준편차를 부풀림 |
| **시각화** | 차트 스케일을 왜곡해 나머지 데이터가 뭉개짐 |

### 이상치 처리 의사결정
1. **이상치의 원인을 알 수 있는가?** → 모르면: 포함/제외 모두 분석해 비교.
2. **입력·측정 오류인가?** → 예: 수정 또는 제거.
3. **분석 대상에 포함되어야 하는 사례인가?** → 예: 제거하지 말고 중앙값 등 강건한 통계 사용 / 아니오: 별도 분석 또는 제거 후 이유를 기록.

- **가장 나쁜 처리**: 이유 없이 삭제. **가장 좋은 처리**: 원인 파악, 포함/제외 모두 분석, 과정을 투명하게 기록.

### 실전 예시
- 수면 "25시간" → 불가능한 값 → **입력 오류**, 제거 또는 재확인.
- 스마트폰 0시간 1명 → 실제로 없을 수 있음 → **유효한 데이터**로 포함, 특이 사례로 주목.
- 크리스마스 당일 매출이 평소 5배 → **시즌 효과** → 분석 목적에 따라 포함/제외 결정.

## 레슨 과제

1. 데이터 표(테이블)에서 '가로줄(행)'과 '세로줄(열)'은 각각 어떤 정보를 담고 있나요?

2. 데이터를 크기순으로 정렬해 보았을 때 가장 편리했던 점 한 가지를 적어주세요.

3. 우리 반 수학 점수를 분석한다고 상상해 봅시다. '평균'과 '중앙값'은 어떤 차이가 있을까요?

4. 산점도, 박스 플롯, 막대 차트 중 가장 기억에 남는 그래프 하나를 골라보세요. 그 그래프를 그리면 데이터의 어떤 점(예: 두 가지의 관계, 데이터가 몰려 있는 곳 등)을 한눈에 알 수 있나요?

5. 그래프를 보다가 다른 데이터들과 혼자 멀리 떨어져서 유독 튀는 '이상치'를 발견했습니다. 이런 이상치는 무조건 지워버려야 하는 쓸모없는 데이터일까요? 이상치를 어떻게 대해야 할지 배운 내용을 바탕으로 적어보세요.

6. '데이터 탐색' 단원을 배우기 전과 후, 데이터를 바라보는 여러분의 생각은 어떻게 달라졌나요?
