## 8-1 ML이란?

### ML이란?
- 머신러닝 = "데이터에서 패턴을 학습하여 새로운 데이터에 대해 예측/판단하는 것"
- 전통적 프로그래밍: 사람이 규칙 + 데이터 → 결과.
- 머신러닝: 데이터 + 결과(정답) → 규칙(모델). 컴퓨터가 데이터에서 규칙을 학습.

### 일상 속 머신러닝
| 예시 | 방식 | 유형 |
|------|------|------|
| 스팸 메일 필터 | 수천 개 스팸/정상 메일 학습 → 새 메일 판단 | 분류 |
| 음악 추천 | 내가 들은 노래 패턴 학습 → 좋아할 만한 노래 추천 | 추천 |
| 검색어 자동완성 | 검색 기록 학습 → 입력 중 나머지 예측 | 예측 |
| 얼굴 인식 | 수만 장 얼굴 학습 → 사진 속 사람 판단 | 분류 |
| 날씨 예측 | 과거 기상 데이터 학습 → 미래 예측 | 예측 |
| 자율주행 | 도로 상황 데이터 학습 → 실시간 운전 판단 | 분류/예측 |

### 규칙 vs 학습 — 언제 ML이 필요할까?
- 숫자 짝/홀 판별 → 규칙이 명확 → 규칙(프로그래밍)이 효율적.
- 사진 속 고양이/개 구분 → 규칙으로 표현 어려움 → ML이 유리 (많은 데이터 필요).
- 내일 비 올 확률 → 변수가 많고 복잡 → ML이 더 정확 (단, 설명 어려움).

### ML의 3가지 유형
| 유형 | 영문 | 설명 | 예시 |
|------|------|------|------|
| 지도학습 | Supervised Learning | 정답(라벨)이 있는 데이터로 학습 | 스팸 분류, 집값 예측, 질병 진단 |
| 비지도학습 | Unsupervised Learning | 정답 없이 데이터의 구조·패턴 발견 | 고객 그룹 분류, 이상 거래 탐지, 주제 분류 |
| 강화학습 | Reinforcement Learning | 시행착오를 통해 보상 최대화하는 행동 학습 | 게임 AI, 로봇 제어, 자율주행 |

- 이 챕터에서는 지도학습에 집중. 다음 레슨에서 분류와 회귀 체험.

## 8-2 분류 체험

### 분류(Classification)
- 데이터를 미리 정해진 카테고리 중 하나로 분류하는 작업. 예: 스팸/정상 메일, 질병/정상 진단, 합격/불합격 예측.

### 의사결정 트리
- "질문을 순서대로 던져서 분류하는" 방법. 스무고개와 비슷.
- 예: 과일 분류 — 색깔이 빨간가? → 예: 크기가 작은가? → 예: 체리 / 아니오: 사과. 아니오: 색이 노란가? → 예: 바나나 / 아니오: 포도.

### 실전: 학생 성적만족도 분류
- 수면 시간과 주당 운동일수를 기준으로 성적 만족도가 "높음"인지 "낮음"인지 분류. 산점도에서 분류 경계를 직접 그려보는 실습.

### 의사결정 트리의 장단점
- 장점: 해석이 쉽다. "수면 6.5시간 이상이면 성적만족도가 높을 확률이 높다"처럼 규칙을 사람이 이해할 수 있음.
- 단점: 데이터가 복잡해지면 트리가 너무 커져 과적합(외우기만 하고 일반화 못 함) 위험.

## 8-3 회귀 체험

### 회귀(Regression)
- 연속적인 수치를 예측하는 ML 기법. 분류가 "A인가 B인가?"라면, 회귀는 "얼마인가?"를 예측.

### 분류 vs 회귀
| 분류 | 회귀 |
|------|------|
| "합격 or 불합격" | "78점? 92점?" |
| 범주(카테고리) 예측 | 숫자(연속값) 예측 |

### 선형 회귀
- 가장 기본적인 회귀. 데이터에 가장 잘 맞는 직선을 찾음. 공식: y = ax + b (a=기울기, b=절편).
- 최소제곱법: 모든 데이터 점과 직선 사이의 거리(오차)를 최소화하는 직선을 찾는 방법.

### 실습
- 직선 피팅 놀이터: 기울기·절편을 조절해 데이터에 맞는 직선 찾기, 자동 피팅과 비교.
- 수면 시간으로 시험 점수 예측: 수면–점수 데이터로 회귀 모델 만들고, 새 수면 시간에 대한 점수 예측.

### 회귀 모델의 한계
- 데이터 범위 밖 예측(외삽)은 위험 (수면 12시간 → 점수 120점?).
- 직선이 항상 맞는 것은 아님 (비선형 관계 존재).
- 상관관계 ≠ 인과관계.
- R²가 높다고 항상 좋은 모델은 아님.

## 8-4 나의 첫 ML 프로젝트

### ML 프로젝트 7단계
| Step | 제목 | 설명 | 예시 |
|------|------|------|------|
| 1 | 문제 정의 | 무엇을 예측/분류할 것인가? | "수면 시간으로 성적만족도를 예측할 수 있을까?" |
| 2 | 데이터 수집 | 필요한 데이터를 모으고 정리 | 설문조사, 공공데이터, 실험 |
| 3 | 데이터 탐색 | 시각화·통계로 데이터 이해 | 산점도, 히스토그램, 이상치 확인 |
| 4 | 데이터 전처리 | 결측치·이상치·변환 | 빈 값 채우기, 스케일링, 범주형 인코딩 |
| 5 | 모델 학습 | 알고리즘 선택 + 학습 데이터로 훈련 | 의사결정 트리, 선형 회귀, 랜덤 포레스트 |
| 6 | 모델 평가 | 테스트 데이터로 성능 측정 | 정확도, R², 혼동행렬 |
| 7 | 해석 & 활용 | 결과 해석·의사결정에 활용 | "수면 7시간 이상이면 만족도 높을 확률 80%" |

- 데이터 준비(1–4단계)가 전체 시간의 70–80%. "쓰레기 데이터로는 쓰레기 모델만 나온다(GIGO)".

### ML 지식 퀴즈 예시
- ML 프로젝트에서 가장 먼저 할 일? → 문제 정의.
- 학습/테스트 데이터를 분리하는 이유? → 모델의 일반화 능력을 평가하기 위해.
- 정확도 100%라면? → 과적합(overfitting) 의심.
- 80%로 학습·20%로 테스트하는 비율? → 8:2 분할(train-test split).

### 미니 ML 프로젝트 설계 예
- 학생 성적 예측: 수면·운동·스마트폰 → 성적만족도(높음/낮음), 의사결정 트리, 200명 설문.
- 기온 예측: 월·연도 → 평균기온(°C), 선형 회귀, 서울 월별 기온 2015–2025.
- 운동과 수면: 주당 운동일수 → 수면 시간(시간), 선형 회귀, 200명 설문.

### 과적합 vs 과소적합
- 과적합: 학습 데이터에만 너무 맞춤 → 새 데이터에서 성능 저하.
- 과소적합: 모델이 너무 단순 → 학습·테스트 모두 성능 낮음.

### ML은 "마법의 도구"가 아님
- 좋은 데이터 없이는 좋은 모델을 만들 수 없음.
- 모델 예측이 항상 맞는 것은 아님.
- ML 결과도 비판적으로 검증해야 함.
- 윤리적 고려 없는 ML은 해로울 수 있음. 데이터 리터러시가 있는 사람만이 ML을 올바르게 활용할 수 있음.
