# 수업자료 비공개 페이지 — 나만 보는 방법

수업자료(md)를 웹에 올려 두되, **학생에게는 노출되지 않고** 교사만 볼 수 있게 하는 방법입니다.

## 1. 어떻게 동작하나요?

- **비밀 경로**: 사이트 어디에도 링크가 없습니다. 메뉴·푸터·홈에 수업자료 링크가 없어서 학생이 클릭해서 들어올 수 없습니다.
- **비밀키**: 올바른 주소로 들어가도, **URL에 비밀키**가 없으면 "페이지를 찾을 수 없습니다"만 보입니다. 비밀키는 본인만 알고 있으면 됩니다.

## 2. 설정 방법

### ① 비밀키 정하기

- 본인만 아는 **영문·숫자 조합**을 정합니다.  
  예: `myTeaching2025`, `goo-internal-키`

### ② 환경 변수 설정

1. 프로젝트 루트에 **`.env`** 파일을 만듭니다 (없다면).
2. 아래 한 줄을 넣고, `비밀키`를 위에서 정한 값으로 바꿉니다.

```env
VITE_TEACHING_KEY=비밀키
```

예:

```env
VITE_TEACHING_KEY=myTeaching2025
```

3. **저장 후** 개발 서버를 다시 실행합니다 (`npm run dev`).  
   배포 후에는 **배포 환경(호스팅)**에서도 같은 이름(`VITE_TEACHING_KEY`)으로 환경 변수를 설정해야 합니다.

### ③ 접속 주소

- **개발**: `http://localhost:4010/goo-teaching?k=비밀키`  
  예: `http://localhost:4010/goo-teaching?k=myTeaching2025`
- **배포 후**: `https://내사이트주소/goo-teaching?k=비밀키`

이 주소를 **북마크**해 두면, 학생에게 알리지 않는 한 학생은 이 페이지를 찾을 수 없습니다.

## 3. 보안에 대해

- **URL을 아무에게나 알리지 마세요.** 이 주소를 알면 비밀키를 아는 사람은 누구나 수업자료를 볼 수 있습니다.
- 비밀키는 **.env**에만 두고, GitHub 등에 올리지 마세요. (`.env`는 보통 `.gitignore`에 들어 있어서 커밋되지 않습니다.)
- 필요하면 **비밀키를 주기적으로 바꾸고**, 새 주소만 본인만 사용하세요.

## 4. 경로를 더 숨기고 싶다면

- `goo-teaching` 대신 **아무도 추측하기 어려운 경로**로 바꿀 수 있습니다.  
  예: `x7k2m9p`, `internal-docs-2025`  
- 수정 위치: `src/App.jsx` 안의  
  `path="goo-teaching"`  
  부분을 원하는 경로로 바꾼 뒤, 접속 시에도 같은 경로를 사용하면 됩니다.

이렇게 하면 학생들은 이 페이지의 존재를 모른 채, 교사만 수업자료를 웹에서 볼 수 있습니다.
