## 1-1 우리 주변의 데이터

### 도입
- 우리는 매일 데이터 속에서 산다. 날씨, 버스 도착 시간, SNS 좋아요 수 등.
- 데이터(Data): 관찰이나 측정을 통해 수집된 사실이나 값. 기록 가능하고 분석 가능한 정보.

### 데이터의 세 가지 특징
1. 측정 가능 — 숫자든 범주든 일정한 기준으로 기록 가능 (예: 키 172cm, 혈액형 A형)
2. 기록 가능 — 누가 언제 보더라도 같은 값 확인 가능 (예: 2026년 2월 22일 서울 기온 -3°C)
3. 분석 가능 — 모아서 패턴·비교·예측에 활용 가능 (예: 10년간 기온 → 온난화 추세)

### 실습: 이것은 데이터일까?
- 기온 -3°C → 데이터 (측정 가능한 수치)
- "오늘 날씨가 좀 추운 것 같아" → 비데이터 (주관적 느낌)
- 급식 만족도 "4점(5점 만점)" → 데이터 (설문 응답, 숫자로 기록)
- "다음 버스: 5분 후" → 데이터 (GPS·운행 데이터 기반 예측)

### 의견을 데이터로
- "맛있다"는 의견이지만, "맛 평점 4.5/5"로 기록하면 데이터가 됨. 기록 방식에 따라 의견도 데이터가 될 수 있음.

### 주변에서 데이터 찾기 (예시)
- 교실: 학생 수, 교실 온도, 출석률, 과목별 시험 점수
- 편의점: 상품 가격, 일일 판매량, 재고, 방문 고객 수, 결제 수단
- 스마트폰: 스크린 타임, 걸음 수, 배터리 사용량, 앱별 사용 시간

### 왜 데이터가 중요한가?
- 감을 근거로 바꾸는 힘. "요즘 학생들이 스마트폰을 너무 많이 하는 것 같아요" → "고등학생 평균 3.5시간, 2년 전보다 0.8시간 증가"처럼 근거 있는 진술로 전환.

## 1-2 데이터의 종류

### 두 가지 큰 분류: 범주형 vs 수치형

| 유형 | 설명 | 예시 |
|------|------|------|
| 범주형 | 이름·그룹으로 분류, 평균·합계 의미 없음 | 혈액형, 성별, 좋아하는 과목, 거주 지역 |
| 수치형 | 숫자로 표현, 계산(평균·합계·비교) 가능 | 키, 시험 점수, 기온, 몸무게 |

### 주의
- 숫자라고 다 수치형이 아님. 학번, 전화번호는 평균/합계가 의미 없으므로 범주형.
- 핵심 질문: "이 숫자들의 평균을 구하면 의미가 있는가?"

### 측정 척도 (더 세밀한 분류)
| 척도 | 설명 | 예시 |
|------|------|------|
| 명목 | 이름만 구분, 순서 없음 | 혈액형, 성별, 지역 |
| 순서 | 순서는 있으나 간격이 균일하지 않음 | 학년, 만족도(매우불만–매우만족) |
| 등간/비율 | 간격 균일, 사칙연산 가능 | 기온, 키, 점수, 시간 |

### 데이터 종류와 분석
- 범주형 → 빈도, 비율, 막대·원형 차트
- 수치형 → 평균, 표준편차, 히스토그램, 산점도
- 잘못된 차트·분석을 쓰면 틀린 결론에 도달할 수 있음.

## 1-3 좋은 데이터 vs 나쁜 데이터

### GIGO 원칙
- Garbage In, Garbage Out — 입력 데이터가 엉망이면 결과도 엉망. 데이터 품질 확인이 첫 번째 단계.

### 데이터 품질 5가지 기준
| 기준 | 질문 | 나쁜 예 |
|------|------|---------|
| 완전성 | 빈 값(결측치)이 없는가? | 빈 칸이 많은 설문 |
| 정확성 | 값이 정확하고 현실적인가? | 나이 -5세, 키 300cm |
| 일관성 | 같은 기준으로 기록되었는가? | 날짜 형식 혼재 (2026/01/01, 1월 2일, Jan 3) |
| 최신성 | 충분히 최근 데이터인가? | 2010년 물가로 2026년 분석 |
| 중복 없음 | 같은 데이터가 반복되지 않는가? | 한 사람이 두 번 기록됨 |

### 문제 유형별 처리
- 결측치: 해당 행 제거, 평균/중앙값으로 채우기, 가장 가까운 값으로 채우기
- 이상값: 명백한 오류면 제거, 의미 있는 이상값이면 별도 분석, 전문가 확인
- 중복: 중복 행 찾아 하나만 남기기, 어떤 값이 맞는지 확인 후 통합
- 일관성: 통일된 형식으로 변환, 오타·약어 표준화

### 결측치 삭제 시 유의
- 200명 중 50명 응답 비어 있으면 삭제 시 150명만 남음. 비어 있는 응답에 패턴이 있을 수 있음(예: 남학생만 미응답). 무작정 삭제보다 왜 비어 있는지 먼저 생각하는 것이 중요.

## 챕터 1 종합: 이번에 배운 것

- 1-1: 데이터 정의(관찰/측정·기록·분석 가능), 3가지 특징(측정·기록·분석), 의견과 데이터의 차이, 데이터가 중요한 이유(감→근거).
- 1-2: 범주형 vs 수치형, 숫자라도 범주형일 수 있음(학번 등), 명목·순서·등간/비율 척도, 데이터 종류에 따른 분석 방법.
- 1-3: GIGO, 품질 5기준(완전성·정확성·일관성·최신성·중복 없음), 품질 문제 진단, 결측·이상값·중복 처리 방법.

## 레슨 과제

1. '데이터'의 세 가지 특징(측정 가능, 기록 가능, 분석 가능) 중 하나를 골라, 우리 일상에서 그 특징을 만족하는 데이터 예를 하나 들어보세요.

2. '혈액형', '시험 점수', '학번'을 범주형과 수치형으로 나누어 보세요. 숫자가 들어 있어도 수치형이 아닐 수 있는 이유는 무엇인가요?

3. GIGO(Garbage In, Garbage Out)가 의미하는 바를 한 문장으로 설명하고, 데이터 품질을 확인해야 하는 이유를 적어보세요.

4. 데이터 품질의 5가지 기준(완전성, 정확성, 일관성, 최신성, 중복 없음) 중에서, 결측치(빈 값)가 많은 데이터는 어떤 기준에서 문제가 있는가요? 그런 데이터를 그대로 분석하면 어떤 일이 생길 수 있을까요?

5. '의견'과 '데이터'는 어떻게 다릅니다? 의견을 데이터로 바꾸려면 어떤 방식으로 기록하면 될까요? 구체적인 예를 들어보세요.

6. '데이터란 무엇인가' 단원을 배운 뒤, 일상에서 '이것도 데이터구나'라고 새로 느낀 것이 있다면 적어보세요.
